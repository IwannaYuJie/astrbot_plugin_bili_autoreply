# 开发计划

## 第一阶段：基础骨架（当前阶段）

目标：把项目结构、设计文档、配置项、基础指令先立住。

已包含：

- 插件元信息
- 基础配置 Schema
- 状态检查命令
- B 站探针命令
- 评论只读扫描命令
- LLM dry run 命令
- 文档

---

## 第二阶段：只读评论链路

计划内容：

- 获取视频列表
- 获取指定视频评论
- 构建评论数据结构
- 提取 `@` 目标与楼层上下文
- 保存最近扫描状态

当前进度：

- 已支持读取最近视频和最新评论
- 已支持按用户名文本匹配 `@你`
- 当前仍为只读，不执行回复

验收标准：

- 能够成功读取自己视频的评论
- 能在结果中看到命中规则的评论
- 不执行回复

---

## 第三阶段：单次手动回复

计划内容：

- 实现发送评论回复接口
- 使用命令手动触发一次回复流程
- 增加最基本的幂等控制

验收标准：

- 指定 comment_id 可成功发送一次回复
- 重复发送被拦截

---

## 第四阶段：自动轮询

计划内容：

- 引入后台轮询任务
- 启动/停止轮询
- 每轮限量处理
- 异常不中断插件

验收标准：

- 自动扫描并回复命中评论
- 插件可重载，不重复创建轮询任务

---

## 第五阶段：上线增强

计划内容：

- Cookie 刷新
- 审核模式
- 黑白名单
- 敏感词拦截
- 更详细日志
- 失败重试与退避

验收标准：

- 连续运行更稳定
- 错误定位明确
- 误回复风险可控
